# Data Ownership Model

`selamnow-dr-automation` does not own persistent business data. It governs the mechanisms that ensure the survivability of that data.

## 1. Managed State
- **Terraform State:** Owns the definitions for regional infrastructure and peering layers.
- **GitOps State:** Owns the declarative application targets for ArgoCD during incident failovers.
- **Failover Logs:** Owns the immutable records of RTO/RPO validation generated by simulated or actual recovery procedures.

## 2. Unmanaged State
- **Ledger Data:** It controls the replication posture for Core Platform but DOES NOT interact with account balances.
- **Identity Data:** Governs synchronous commit settings for the database cluster but DOES NOT access user credentials.
- **Payment Payloads:** Orchestrates the clusters that clear transactions, but DOES NOT process the payloads themselves.

Data privacy and residency boundaries are strictly maintained during automated failover sequences.
